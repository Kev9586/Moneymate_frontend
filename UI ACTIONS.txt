1 — Splash Screen (file: src/screens/Splash.tsx)
What to build
• Full-screen gradient background, centered LogoMark.
• Reanimated sequence: fade-in 0→1 (400ms), scale 0.96→1 (300ms spring).
How it behaves
• On mount call await auth.bootstrap():
• Read firstRun and token from MMKV.
• If firstRun === true → navigation.replace('Onboarding').
• Else if token exists → navigation.replace('MainTabs').
• Else → navigation.replace('Auth').
• If NetInfo offline → still follow stored-state routing.
Edge
• If app backgrounds during animation, resume routing on AppState change.
• Cancel animation safely on unmount.
Tests
• Unit: mount with fake MMKV states → assert navigation target.
• Detox: cold start flows.

3 — Authentication (files: src/screens/Auth.tsx, src/modals/OtpModal.tsx)
What to build
• Tabbed UI: Login | Signup using Segmented control.
• Inputs: FormInput.tsx, FormPassword.tsx (React Hook Form + Zod).
• OTP modal works for Email/Phone, Signup, Forgot Password, 2FA.
Actions
• Login
• Validate → secureFetch('/auth/login', POST) → on success auth.setUser() & auth.setToken() → navigation.replace('MainTabs').
• Signup
• POST /auth/signup/init → open OTP modal → verify OTP → /auth/signup/complete → store token → navigate('MainTabs').
• Forgot Password
• OTP modal bound to Email, verify → /auth/reset.
Animations
• Shared horizontal slide for tab transitions; primary button bounce withSpring.
Edge
• Invalid creds → toast + input shake.
• No network → ErrorFullScreen with retry to re-submit last action.
• Disable submit while pending.
Tests
• Unit: form validation schemas.
• Integration: mock success and 401.
• Detox: signup + OTP flow (staging/mock).
